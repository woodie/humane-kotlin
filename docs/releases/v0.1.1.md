# humane-kotlin v0.1.1

## Summary

No behavior change to `Humane.humanSize`/`distanceInTime`/`timeAgo` -- this
release is entirely build/tooling/packaging. It bundles up everything that
landed on `main` since `v0.1.0` plus this repo's first Maven Central
publishing setup.

## Since v0.1.0

- Switched off the `kotidy` composite build (`pluginManagement {
  includeBuild("../kotidy") }`) now that `com.netpress.kotidy` is approved
  and live on the Gradle Plugin Portal -- pins `id("com.netpress.kotidy")
  version "0.1.0"` instead, resolved via `gradlePluginPortal()`. CI's
  sibling-checkout step for `kotidy` is gone too.
- Added `.github/workflows/CI.yml`.
- `make format`/`ktlintFormat`-then-build ordering fix, matching huck's own
  Makefile.
- README project badges.

## Maven Central publishing (new)

Applied [`com.vanniktech.maven.publish`](https://vanniktech.github.io/gradle-maven-publish-plugin/central/)
0.37.0 -- `publishToMavenCentral()` + `signAllPublications()`, coordinates
`com.netpress:humane-kotlin`, and the required POM fields (license, SCM,
developer). GPG signing is new infrastructure for this account; see
[issue #1](https://github.com/woodie/humane-kotlin/issues/1) for the full
Central Portal setup (namespace verification, user token, GPG keypair).

First publish under this version uses manual publishing (`publishToMavenCentral()`
with no arguments) -- the deployment lands in "Validated" state on the
[Central Portal](https://central.sonatype.com/publishing/deployments) for a
manual "Publish" click, rather than releasing automatically.

## Scope

Still consumed by [`huck`](https://github.com/woodie/huck) via
`includeBuild("../humane-kotlin")` -- switching `huck` to the published
Maven Central artifact is tracked separately in issue #1, once this release
is confirmed live.
