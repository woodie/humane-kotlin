# humane-kotlin v0.1.0

## Initial release

Fourth-language sibling of [`humane`](https://github.com/woodie/humane) (Go),
[`humane-ruby`](https://github.com/woodie/humane-ruby), and
[`humane-swift`](https://github.com/woodie/humane-swift) -- same algorithm,
wording, and fixtures, ported to Kotlin/JVM. `HumanSize`, `DistanceInTime`,
and `TimeAgo` format file sizes and relative dates the way macOS Finder does.

## Usage

```kotlin
import com.netpress.humane.Humane

Humane.humanSize(225_935)                 // "226 KB"
Humane.distanceInTime(at, Instant.now())  // "3 minutes ago"
Humane.timeAgo(downloadedAt)              // "3 minutes ago"
```

## One deliberate API difference from Ruby/Swift

`timeAgo`'s `at` parameter is non-nullable, matching `humane`'s Go `v0.9.4`
decision rather than `humane-ruby`/`humane-swift`'s nullable version --
every real caller has a concrete timestamp, so nil-handling stays with
`distanceInTime`'s explicit core rather than the one-argument convenience.
See `docs/COMMENTS.md`.

## Testing

All 42 Kotest `DescribeSpec` examples across `HumanSizeSpec`/
`DistanceInTimeSpec`/`TimeAgoSpec` pass, confirmed on real hardware --
including every approximate-distance boundary-table cutoff (44:29/44:30,
89:29/89:30, 23:59:29/23:59:30). Kotest renders the real nested
`describe`/`context`/`it` tree natively; no `gorderly`-equivalent was
needed to get there, unlike Go.

## Scope

Built specifically as [`huck`](https://github.com/woodie/huck)'s size/time
formatting dependency, consumed via a Gradle composite build rather than a
published artifact -- not published to Maven Central or GitHub Packages at
this point. `make lint` (`ktlint`) hasn't been run against this codebase yet.
